Nexus Task / Celery 数据流

当前版本没有 startup queued 补发。任务创建成功的定义是 PostgreSQL 记录创建成功，并且 Celery broker publish 成功。worker 丢失后的恢复交给 Celery redelivery，执行侧只根据 task 状态和 checkpoint 决定继续还是跳过。

Details

    当前不变量
    • queued 不会在 startup 自动补发。
    • broker publish 失败必须进入 failed，并向调用方返回错误。
    • running 的恢复只接受 Celery redelivery。
    • checkpoint 是 worker 重启后判断继续或跳过的依据。
